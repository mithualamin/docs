## Overview

HTFlow CSS rules ensure your styles are compatible with Webflow and Droip conversion processes. These rules focus on consistent styling, responsive design, and maintaining clean, maintainable CSS.

<Info>
  Follow these CSS rules to ensure seamless conversion and maintain design
  consistency across your website.
</Info>

## Required CSS Structure

Every HTFlow project must include the `data-ht-styles` attribute:

```html
<style data-ht-styles>
  /* All CSS goes here */
</style>
```

## Container System

### Single Container Pattern

Use consistent container classes across all sections:

```css
/* Main container */
.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 20px;
}

.container-large {
  max-width: 1400px;
  margin: 0 auto;
  padding: 0 20px;
}

.container-small {
  max-width: 800px;
  margin: 0 auto;
  padding: 0 20px;
}
```

### Usage in HTML

```html
<div class="htflow-wrapper">
  <section class="hero-section">
    <div class="container">
      <!-- Hero content -->
    </div>
  </section>

  <section class="features-section">
    <div class="container">
      <!-- Features content -->
    </div>
  </section>

  <section class="testimonials-section">
    <div class="container">
      <!-- Testimonials content -->
    </div>
  </section>
</div>
```

## Layout Systems

### CSS Grid Layouts

```css

  grid-template-columns: repeat(2, 1fr);
  gap: 2rem;
```

## Responsive Design

### Mobile-First Approach

Always start with mobile styles and add larger breakpoints:

```css
/* Mobile first (default) */
.hero-section {
  padding: 2rem 0;
  text-align: center;
}

.hero-title {
  font-size: 2rem;
  margin-bottom: 1rem;
}

/* Tablet */
@media (min-width: 768px) {
  .hero-section {
    padding: 4rem 0;
  }

  .hero-title {
    font-size: 3rem;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .hero-section {
    padding: 6rem 0;
  }

  .hero-title {
    font-size: 4rem;
  }
}

/* Large desktop */
@media (min-width: 1280px) {
  .hero-section {
    padding: 8rem 0;
  }
}
```

### Breakpoint System

```css
/* Standard breakpoints */
@media (min-width: 640px) {
  /* sm */
}
@media (min-width: 768px) {
  /* md */
}
@media (min-width: 1024px) {
  /* lg */
}
@media (min-width: 1280px) {
  /* xl */
}
@media (min-width: 1536px) {
  /* 2xl */
}
```

## Common Mistakes to Avoid

<Warning>
**Never do these things:**

- Use IDs for styling (`#my-element`)
- Use inline styles (`style="color: red"`)
- Use `!important` unless absolutely necessary
- Skip the `data-ht-styles` attribute
- Use different container classes in different sections
- Use inconsistent heading sizes across pages
- Forget responsive design (mobile-first)
- Use emojis instead of SVG icons
- Use `float` for layouts (use Grid/Flexbox)
- Skip proper spacing systems
- Use hardcoded colors instead of CSS variables
- Forget to test on different screen sizes
  </Warning>
