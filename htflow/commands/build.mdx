## Overview

The HTFlow build command creates optimized, production-ready HTML, CSS, and JavaScript files that are ready to be converted and deployed to Webflow.

<Info>
Always run validation and audit before building for production
</Info>

## Build Output Structure

The build creates a `dist` folder with production-ready files:

```text Build Output
dist/
├── index.html              # Optimized HTML
├── about.html
├── styles/
│   ├── main.css           # Minified CSS
│   └── main.css.map       # Source map
├── scripts/
│   ├── main.js            # Bundled & minified JS
│   └── main.js.map        # Source map
├── assets/
│   ├── images/
│   │   ├── hero.jpg       # Optimized images
│   │   └── logo.svg
│   └── fonts/
│       └── custom-font.woff2
└── webflow-manifest.json  # Webflow conversion manifest
```

## Build Configuration

Customize build behavior in your config:

```javascript htflow.config.js
module.exports = {
  build: {
    // Output directory
    outDir: "./dist",
    
    // Minification
    minify: {
      html: true,
      css: true,
      javascript: true
    },
    
    // Source maps
    sourcemap: true,  // Set false for production
    
    // Code splitting
    splitting: true,
    
    // Asset handling
    assets: {
      images: {
        optimize: true,
        formats: ["webp", "jpg", "png"],
        quality: 85
      },
      fonts: {
        subset: true,
        formats: ["woff2", "woff"]
      }
    },
    
    // Cache busting
    hash: true,  // Add hash to filenames
    
    // Webflow-specific
    webflow: {
      manifest: true,
      compatibility: "strict",
      embedReady: true
    },
    
    // Browser targets
    targets: {
      browsers: [
        "last 2 versions",
        "> 1%",
        "not dead"
      ]
    }
  }
};
```

## Build Modes

HTFlow supports different build modes:

<Tabs>
  <Tab title="Production (Default)">
    Fully optimized build for deployment:
    
    ```bash
    htflow build
    # or
    htflow build --mode=production
    ```
    
    Features:
    - Full minification
    - No source maps
    - Asset optimization
    - Cache busting
  </Tab>
  
  <Tab title="Development">
    Faster builds for testing:
    
    ```bash
    htflow build --mode=development
    ```
    
    Features:
    - No minification
    - Source maps included
    - Faster build time
    - Better debugging
  </Tab>
  
  <Tab title="Preview">
    Build for preview/staging:
    
    ```bash
    htflow build --mode=preview
    ```
    
    Features:
    - Light minification
    - Source maps included
    - Asset optimization
    - Fast builds
  </Tab>
</Tabs>

## Build Performance

Optimize your build times:

<AccordionGroup>
  <Accordion title="Reduce File Sizes" icon="compress">

    - Optimize images before building
    - Minimize dependencies
    - Use code splitting
  </Accordion>
  
  <Accordion title="Parallel Processing" icon="gauge">

  <Accordion title="Caching" icon="database">

  <Accordion title="Incremental Builds" icon="rotate">

      incremental: true,  // Only rebuild changed files

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build Fails" icon="circle-xmark">
    **Common causes:**
    - Validation errors
    - Missing dependencies
    - Syntax errors
    
    **Solutions:**
    ```bash
    # Validate first
    htflow validate
    
    # Install dependencies
    npm install
    
    # Clean and rebuild
    rm -rf dist
    htflow build
    ```
  </Accordion>
  
  <Accordion title="Large Bundle Size" icon="weight-hanging">
    **Reduce size:**
    
    ```javascript
    build: {
      minify: true,
      splitting: true,
      assets: {
        images: {
          quality: 75  // Lower quality
        }
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Missing Files in Build" icon="file-circle-xmark">
    **Check:**
    - Files are in source directory
    - Not excluded in `.gitignore`
    - Not in `ignore` patterns
    
    ```javascript
    build: {
      ignore: [
        "**/*.md",
        "**/node_modules/**"
      ]
    }
    ```
  </Accordion>
  
  <Accordion title="Build Takes Too Long" icon="clock">
    **Speed up builds:**
    
    ```javascript
    build: {
      parallel: true,
      cache: true,
      sourcemap: false,  // Disable source maps
      assets: {
        images: {
          optimize: false  // Skip image optimization
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Build Scripts

Integrate builds into your workflow:

<CodeGroup>

```json package.json
{
  "scripts": {
    "build": "htflow build",
    "build:dev": "htflow build --mode=development",
    "build:preview": "htflow build --mode=preview",
    "prebuild": "htflow validate",
    "postbuild": "htflow audit"
  }
}
```

```bash npm
# Run build script
npm run build

# With validation
npm run prebuild && npm run build
```

</CodeGroup>
