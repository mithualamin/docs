## Overview

HTFlow supports MCP (Model Context Protocol) integration, allowing AI assistants like Claude to understand and work with your HTFlow projects directly.

<Info>
MCP integration enables AI assistants to help with HTFlow commands, validation, and project management
</Info>

## What is MCP?

Model Context Protocol (MCP) is a standardized way for AI assistants to interact with tools and systems. With HTFlow MCP integration, AI assistants can:

<CardGroup cols={2}>
  <Card title="Understand Projects" icon="brain">
    Analyze HTFlow project structure and configuration
  </Card>
  <Card title="Run Commands" icon="terminal">
    Execute HTFlow CLI commands on your behalf
  </Card>
  <Card title="Validate Code" icon="shield-check">
    Check code compliance and suggest fixes
  </Card>
  <Card title="Provide Guidance" icon="lightbulb">
    Offer context-aware HTFlow best practices
  </Card>
</CardGroup>

## Install MCP Integration

Set up MCP support for HTFlow:

<Tabs>
  <Tab title="Extension UI">
    <Steps>
      <Step title="Open HTFlow Panel">
        Press `Ctrl+Shift+H` / `Cmd+Shift+H` or click the HTFlow icon
      </Step>
      
      <Step title="Click MCP Install">
        Click the **"Install MCP Integration"** button in the MCP section
      </Step>
      
      <Step title="Configure AI Assistant">
        Follow prompts to configure your AI assistant:
        - Select AI tool (Claude, GPT, etc.)

        Restart your AI assistant to load HTFlow MCP support

## Configuration

MCP configuration is stored in your AI assistant's config:

<CodeGroup>

```json Claude Desktop (claude_desktop_config.json)
{
  "mcpServers": {
    "htflow": {
      "command": "htflow",
      "args": ["mcp"],
      "env": {
        "HTFLOW_PROJECT_PATH": "/path/to/project"
      }
    }
  }
}
```

```json VS Code Settings (settings.json)
{
  "htflow.mcp.enabled": true,
  "htflow.mcp.aiAssistant": "claude",
  "htflow.mcp.permissions": {
    "readProject": true,
    "runCommands": true,
    "validateCode": true
  }
}
```

</CodeGroup>

## MCP Permissions

Control what AI assistants can access:

```javascript htflow.config.js
module.exports = {
  mcp: {
    enabled: true,
    
    permissions: {
      // Project access
      readProject: true,
      writeProject: false,  // Requires approval
      
      // Command execution
      runCommands: true,
      dangerousCommands: false,  // No destructive ops
      
      // Code access
      readCode: true,
      modifyCode: false,  // Requires approval
      
      // Build access
      build: true,
      deploy: false  // Requires approval
    },
    
    // Allowed commands
    allowedCommands: [
      "validate",
      "audit",
      "build",
      "dev"
    ],
    
    // Blocked commands
    blockedCommands: [
      "uninstall"
    ]
  }
};
```

## Uninstall MCP Integration

Remove MCP support:

<Tabs>
  <Tab title="Extension UI">
    <Steps>
      <Step title="Open HTFlow Panel">
        Press `Ctrl+Shift+H` / `Cmd+Shift+H`
      </Step>
      
      <Step title="Click MCP Uninstall">
        Click the **"Uninstall MCP Integration"** button
      </Step>
      
      <Step title="Confirm Removal">
        Confirm the uninstallation
        
        <Warning>This will remove HTFlow MCP from your AI assistant</Warning>
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Command Line">
    ```bash
    htflow mcp-uninstall
    ```
    
    Output:
    ```text

    âœ“ Removed from AI assistant config
    âœ“ Cleaned up permissions
    âœ“ Removed context data
    
    MCP Integration uninstalled.
    Restart your AI assistant to complete removal.
    ```
  </Tab>
</Tabs>

## AI Assistant Setup Guides

<Tabs>
  <Tab title="Claude Desktop">
    <Steps>
      <Step title="Install HTFlow MCP">
        Run `htflow mcp-install`
      </Step>
      
      <Step title="Locate Config">
        Find your Claude config file:
        - **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
        - **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
      </Step>
      
      <Step title="Verify Configuration">
        Ensure HTFlow is listed in `mcpServers`
      </Step>
      
      <Step title="Restart Claude">
        Restart Claude Desktop app
      </Step>
      
      <Step title="Test Integration">
        Ask Claude: "What HTFlow commands are available?"
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Cursor AI">
    <Steps>
      <Step title="Install HTFlow Extension">
        Ensure HTFlow extension is installed in Cursor
      </Step>
      
      <Step title="Enable MCP">
        Run `htflow mcp-install` in Cursor terminal
      </Step>
      
      <Step title="Configure Settings">
        Add to Cursor settings:
        ```json
        {
          "htflow.mcp.enabled": true
        }
        ```
      </Step>
      
      <Step title="Test Integration">
        Ask Cursor AI to validate your HTFlow project
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Custom Setup">
    For other MCP-compatible AI assistants:
    
    ```json MCP Server Config
    {
      "mcpServers": {
        "htflow": {
          "command": "htflow",
          "args": ["mcp"],
          "env": {
            "HTFLOW_PROJECT_PATH": "${workspaceFolder}"
          },
          "permissions": {
            "filesystem": "read",
            "commands": "execute"

        }

    }
    ```

## Troubleshooting

<AccordionGroup>
  <Accordion title="MCP Not Working" icon="circle-xmark">
    **Solutions:**
    
    1. Verify installation: `htflow version`
    2. Check AI assistant config
    3. Restart AI assistant
    4. Reinstall MCP: `htflow mcp-uninstall && htflow mcp-install`
  </Accordion>
  
  <Accordion title="Permission Denied" icon="lock">
    **Fix permissions:**
    
    ```javascript
    mcp: {
      permissions: {
        runCommands: true,  // Enable
        readProject: true
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Commands Not Found" icon="magnifying-glass">
    **Verify:**
    - HTFlow CLI is installed globally
    - PATH includes npm global bin
    - AI assistant has command permissions
  </Accordion>
</AccordionGroup>
