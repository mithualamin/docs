## Overview

The HTFlow development server provides a local environment with hot reloading, automatic port management, and real-time preview of your Webflow-compatible project.

<Info>
Make sure you've [initialized a project](/htflow/commands/project-management) before starting the development server.
</Info>

## Start Development Server

Launch a local development server for your HTFlow project.

<Tabs>
  <Tab title="Extension UI">
    <Steps>
      <Step title="Open Your Project">
        Make sure you have an HTFlow project open in VS Code
      </Step>
      
      <Step title="Open HTFlow Panel">
        Press `Ctrl+Shift+H` / `Cmd+Shift+H` or click the HTFlow icon
      </Step>
      
      <Step title="Click Start Dev Server">
        Click the **"Start Development Server"** button
        
        <Check>The server will start automatically with smart port handling</Check>
      </Step>
      
      <Step title="View in Browser">
        Your default browser will open automatically to:
        ```text
        http://localhost:3000
        ```
        
        <Tip>The extension displays running servers in the HTFlow panel</Tip>
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Command Line">
    ```bash
    htflow dev
    ```
    
    Output example:
    ```text

    ‚úì Server running at http://localhost:3000
    ‚úì Hot reloading enabled
    ‚úì Press Ctrl+C to stop
    ```

  </Tab>
</Tabs>

## Hot Reloading

HTFlow automatically reloads your changes:

<Tabs>
  <Tab title="HTML Changes">
    Edit any HTML file and see changes instantly:
    
    ```html index.html
    <div class="hero">
        <h1>Updated Title</h1>
        <!-- Changes appear immediately -->
    </div>
    ```
    
    <Check>Full page reload for HTML changes</Check>
  </Tab>
  
  <Tab title="CSS Changes">
    CSS updates without page reload:
    
    ```css styles/main.css
    .hero {
        background: #16A34A; /* Updates instantly */
    }
    ```
    
    <Check>Hot module replacement for CSS</Check>
  </Tab>
  
  <Tab title="JavaScript Changes">
    JavaScript updates with module replacement:
    
    ```javascript scripts/main.js
    console.log('Updated code'); // Runs immediately
    ```
    
    <Check>Smart reload preserves application state</Check>
  </Tab>
</Tabs>

## Port Management

HTFlow intelligently manages ports:

### Default Port

```javascript htflow.config.js
module.exports = {
  dev: {
    port: 3000,  // Default port
    host: "localhost"
  }
};
```

If port 3000 is busy, HTFlow automatically finds the next available port:

## Stop Development Server

Stop the running development server:

<Tabs>
  <Tab title="Extension UI">
    <Steps>
      <Step title="Open HTFlow Panel">
        Press `Ctrl+Shift+H` / `Cmd+Shift+H`
      </Step>
      
      <Step title="Click Stop Server">
        Click the **"Stop Server"** button next to the running server
      </Step>
      
      <Step title="Confirm">
        The server will stop and be removed from the running servers list
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Command Line">
    Press `Ctrl+C` in the terminal where the server is running
    
    ```text
    ^C
    ‚úì Server stopped
    ‚úì Cleanup complete
    ```
  </Tab>
</Tabs>

## Development Configuration

Customize your development environment:

```javascript htflow.config.js
module.exports = {
  dev: {
    // Server settings
    port: 3000,
    host: "localhost",
    
    // Auto-open browser
    open: true,
    
    // Hot reloading
    hot: true,
    
    // Live reload on errors
    liveReload: true,
    
    // Error overlay
    overlay: {
      warnings: true,
      errors: true
    },
    
    // Compression
    compress: true,
    
    // HTTPS (optional)
    https: false,
    
    // Proxy (optional)
    proxy: {
      "/api": "http://localhost:8080"
    }
  }
};
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Server Won't Start" icon="circle-xmark">
    **Common issues:**
    - Port already in use
    - Project not initialized
    - Missing dependencies
    
    **Solutions:**
    ```bash
    # Check if port is in use
    lsof -i :3000  # macOS/Linux
    netstat -ano | findstr :3000  # Windows
    
    # Install dependencies
    npm install
    
    # Reinitialize project
    htflow init
    ```
  </Accordion>
  
  <Accordion title="Hot Reload Not Working" icon="fire-flame-simple">
    **Fixes:**
    1. Check console for errors
    2. Clear browser cache
    3. Restart the development server
    4. Verify `hot: true` in config
    
    ```javascript
    // Ensure hot reloading is enabled
    dev: {
      hot: true,
      liveReload: true
    }
    ```
  </Accordion>
  
  <Accordion title="Can't Access from Network" icon="wifi">
    **Enable network access:**
    
    ```javascript
    dev: {
      host: "0.0.0.0",  // Listen on all interfaces
      port: 3000
    }
    ```
    
    <Warning>Only enable network access on trusted networks</Warning>
  </Accordion>
  
  <Accordion title="Browser Won't Open" icon="window">
    **Solutions:**
    1. Manually open the URL shown in the output
    2. Disable auto-open in config:
    
    ```javascript
    dev: {
      open: false
    }
    ```
    
    3. Check default browser settings
  </Accordion>
  
  <Accordion title="Performance Issues" icon="gauge">
    **Optimization steps:**
    - Reduce file sizes
    - Disable unnecessary features
    - Check for memory leaks
    - Update HTFlow CLI
    
    ```javascript
    dev: {
      overlay: false,  // Disable if causing issues
      compress: true   // Enable compression
    }
    ```
  </Accordion>
</AccordionGroup>

## Advanced Usage

### Custom Port

Specify a custom port:

```bash
htflow dev --port=8080
```

### HTTPS Development

Enable HTTPS for local development:

```javascript htflow.config.js
module.exports = {
  dev: {
    https: true,
    // Or with custom certificates
    https: {
      key: './certs/server.key',
      cert: './certs/server.crt'
    }
  }
};
```

### Environment Variables

Use environment-specific settings:

```javascript
module.exports = {
  dev: {
    port: process.env.PORT || 3000,
    host: process.env.HOST || "localhost"
  }
};
```
