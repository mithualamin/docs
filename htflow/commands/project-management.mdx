## Overview

HTFlow project management commands help you create and configure new HTFlow projects with the proper structure for Webflow compatibility.

<Info>
Make sure you have [installed HTFlow CLI](/htflow/commands/package-management) before using project management commands.
</Info>

## Initialize Project

Create a new HTFlow project with the necessary structure and configuration files.

<Tabs>
  <Tab title="Extension UI">
    <Steps>
      <Step title="Open Project Folder">
        Open the folder where you want to create your HTFlow project
        - Go to **File → Open Folder**
        - Select an empty folder or create a new one
      </Step>
      
      <Step title="Open HTFlow Panel">
        Press `Ctrl+Shift+H` / `Cmd+Shift+H` or click the HTFlow icon
      </Step>
      
      <Step title="Click Initialize">
        Click the **"Initialize Project"** button in the Project Management section
      </Step>
      
      <Step title="Configure Project">
        Follow the prompts to configure your project:
        - Project name
        - Project description
        - Template selection (if available)
        - Configuration options
        
        <Check>HTFlow will create all necessary files and folders</Check>
      </Step>
    </Steps>
  </Tab>
  
  <Tab title="Command Line">
    ```bash
    htflow init
    ```
    
    Interactive prompts will guide you through setup:
    
    ```text
    ? Project name: my-webflow-project
    ? Description: My awesome Webflow site
    ? Template: (Use arrow keys)
    ❯ Basic
      Landing Page
      Portfolio
      E-commerce
    ```
  </Tab>
</Tabs>

## Project Structure

When you initialize a project, HTFlow creates the following structure:

```text Project Structure
my-project/
├── index.html          # Main HTML file
├── styles/
│   ├── main.css       # Main stylesheet
│   └── components/    # Component styles
├── scripts/
│   ├── main.js        # Main JavaScript
│   └── components/    # Component scripts
├── assets/
│   ├── images/        # Image files
│   └── fonts/         # Font files
├── htflow.config.js   # HTFlow configuration
└── package.json       # Project dependencies
```

## Configuration File

The `htflow.config.js` file controls project behavior:

```javascript htflow.config.js
module.exports = {
  // Project metadata
  name: "my-webflow-project",
  version: "1.0.0",
  
  // Development server settings
  dev: {
    port: 3000,
    host: "localhost",
    hot: true,           // Hot reloading
    open: true           // Auto-open browser
  },
  
  // Build settings
  build: {
    outDir: "./dist",
    minify: true,
    sourcemap: false
  },
  
  // Validation settings
  validation: {
    html: true,
    css: true,
    javascript: true,
    strict: false        // Strict mode
  },
  
  // Webflow compatibility
  webflow: {
    version: "latest",
    compatibility: "strict"
  }
};
```

<Tip>
You can customize these settings after initialization to match your workflow preferences.
</Tip>

## Project Files

### HTML Files

HTFlow creates Webflow-compatible HTML:

```html index.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My HTFlow Project</title>
    
    <!-- Webflow-compatible meta tags -->
    <meta name="generator" content="HTFlow">
    
    <!-- Stylesheets -->
    <link rel="stylesheet" href="styles/main.css">
</head>
<body>
    <!-- Your content here -->
    <div class="container">
        <h1>Welcome to HTFlow</h1>
    </div>
    
    <!-- Scripts -->
    <script src="scripts/main.js"></script>
</body>
</html>
```

### CSS Files

Organized, maintainable styles:

```css styles/main.css
/* Global styles */
:root {
    --primary-color: #16A34A;
    --secondary-color: #07C983;
    --text-color: #333;
}

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

body {
    font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
    color: var(--text-color);
    line-height: 1.6;
}

/* Container */
.container {
    max-width: 1200px;
    margin: 0 auto;
    padding: 20px;
}
```

### JavaScript Files

Clean, modular JavaScript:

```javascript scripts/main.js
// HTFlow Project
console.log('HTFlow project initialized');

// Your code here
document.addEventListener('DOMContentLoaded', () => {
    console.log('DOM loaded');
});
```

## Customizing Your Project

After initialization, you can customize your project:

<Steps>
  <Step title="Update Configuration">
    Edit `htflow.config.js` to adjust settings
  </Step>
  
  <Step title="Add Dependencies">

    ```bash
    npm install package-name
    ```
  </Step>
  
  <Step title="Organize Files">
    Create additional folders and files as needed
  </Step>
  
  <Step title="Configure Build">
    Adjust build settings for your deployment needs
  </Step>
</Steps>

## Best Practices

<AccordionGroup>
  <Accordion title="Project Structure" icon="folder-tree">
    - Keep files organized in logical folders
    - Use clear, descriptive file names
    - Separate concerns (HTML, CSS, JS)
    - Group related components together
  </Accordion>
  
  <Accordion title="Configuration" icon="gear">
    - Review default config settings
    - Adjust dev server port if needed
    - Enable source maps during development
    - Use strict validation for production
  </Accordion>
  
  <Accordion title="Version Control" icon="code-branch">
    - Initialize git repository
    - Add `.gitignore` for node_modules
    - Commit after initialization
    - Use meaningful commit messages
  </Accordion>
  
  <Accordion title="Documentation" icon="book">
    - Document custom configurations
    - Keep README updated
    - Note any Webflow-specific requirements
    - Track dependency versions
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Initialization Fails" icon="circle-xmark">
    **Common issues:**
    - Folder is not empty
    - Insufficient permissions
    - HTFlow CLI not installed
    
    **Solutions:**
    - Use an empty folder
    - Check folder permissions
    - Verify CLI installation: `htflow version`
  </Accordion>
  
  <Accordion title="Config File Errors" icon="file-circle-xmark">
    If `htflow.config.js` has errors:
    
    1. Check for syntax errors
    2. Validate JSON structure
    3. Use default config as reference
    4. Reinitialize if necessary
  </Accordion>
  
  <Accordion title="Template Not Found" icon="magnifying-glass">
    If template selection fails:
    
    1. Update HTFlow CLI: `htflow update`
    2. Try basic template first
    3. Manually copy template files if needed
  </Accordion>
</AccordionGroup>
