## Installation

<Steps>
  <Step title="Install Cursor">
    Download and install Cursor from [cursor.sh](https://cursor.sh/)
    
    <Tip>Cursor automatically imports your VS Code settings and extensions</Tip>
  </Step>
  
  <Step title="Open Extensions">
    Press `Cmd+Shift+X` (Mac) or `Ctrl+Shift+X` (Windows/Linux)
  </Step>
  
  <Step title="Search for HTFlow">
    Type **"Htflow Html to Webflow"** in the search bar
  </Step>
  
  <Step title="Install Extension">
    Click **Install** on the HTFlow extension
    
    <Check>Extension is compatible with Cursor out of the box</Check>
  </Step>
  
  <Step title="Reload Cursor">
    Reload the window to activate HTFlow
    
    HTFlow icon appears in the Activity Bar
  </Step>
</Steps>

## Cursor AI Configuration

Optimize Cursor AI for HTFlow development:

### AI Rules for HTFlow

Create `.cursorrules` file in your project root:

```text .cursorrules
# HTFlow Development Rules

## Goal
Generate HTML, CSS, and JavaScript that is Webflow-compatible and follows HTFlow standards.

## HTML Guidelines
- Use semantic HTML5 elements
- Proper document structure with doctype, html, head, and body
- Include viewport meta tag for responsiveness
- Use descriptive class names (BEM style preferred)
- Add alt text to all images
- Ensure accessibility (ARIA labels where needed)

## CSS Guidelines
- Mobile-first responsive design
- Use flexbox and CSS Grid for layouts
- Avoid fixed positioning where possible
- Use CSS custom properties for theming
- Browser-compatible properties only
- Include vendor prefixes when needed
- Webflow-friendly class naming conventions

## JavaScript Guidelines
- Use vanilla JavaScript (ES5/ES6)
- Avoid external dependencies
- Keep code simple and readable
- Use event delegation
- Ensure browser compatibility
- Comment complex logic

## Webflow Compatibility
- No external frameworks (React, Vue, etc.)
- No build tools required in final output
- All styles in CSS files (no CSS-in-JS)
- Standard HTML form elements
- Webflow-compatible interactions

## Testing
- Validate with HTFlow before converting
- Test on mobile, tablet, and desktop
- Check all interactive elements
- Ensure fast load times

## After Generation
- Run `htflow validate` to check compliance
- Use `htflow dev` to preview
- Run `htflow audit` before building
- Build with `htflow build` for production
```

### Cursor Settings for HTFlow

Configure Cursor for optimal HTFlow development:

```json settings.json
{
  // Cursor AI Settings
  "cursor.ai.model": "claude-3.5-sonnet",
  "cursor.ai.followCursorRules": true,
  
  // HTFlow Extension Settings
  "htflow.validation.validateOnSave": true,
  "htflow.dev.openBrowser": true,
  "htflow.build.showNotifications": true,
  
  // Editor Settings
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  
  // File Associations
  "files.associations": {
    "*.html": "html",
    "*.css": "css",
    "htflow.config.js": "javascript"
  }
}
```

### Step 2: Validate Instantly

After AI generates code:

<Steps>
  <Step title="Auto-Validate">
    HTFlow automatically validates on save (if enabled)
    
    ```json Enable Auto-Validation
    {
      "htflow.validation.validateOnSave": true
    }
    ```
  </Step>
  
  <Step title="Manual Validation">
    Click **"Validate Project"** in HTFlow panel
    
    Or press `Cmd+Shift+Alt+V` / `Ctrl+Shift+Alt+V`
  </Step>
  
  <Step title="Fix Issues with AI">
    If validation shows errors:
    
    ```text AI Prompt
    "Fix these HTFlow validation errors:
    [paste errors from Output panel]
    
    Make the code Webflow-compatible"
    ```
    
    AI fixes issues automatically
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cursor Not Using .cursorrules" icon="file">
    **Solution:**
    1. Ensure file is named `.cursorrules` (with dot)
    2. Place in project root directory
    3. Enable in Cursor settings: `cursor.ai.followCursorRules: true`
    4. Reload window
  </Accordion>
  
  <Accordion title="AI Generates Non-Compatible Code" icon="code">
    **Fix:**
    1. Update `.cursorrules` with more specific guidelines
    2. Always mention "Webflow-compatible" in prompts
    3. Use validation after generation
    4. Ask AI to fix issues found by HTFlow
  </Accordion>
  
  <Accordion title="Extension Conflicts" icon="triangle-exclamation">
    **Check:**
    - Disable conflicting extensions
    - HTFlow works best as primary HTML/CSS tool
    - Keep Cursor AI extensions enabled
  </Accordion>
  
  <Accordion title="MCP Not Working" icon="plug">
    **Solutions:**
    1. Reinstall MCP: `htflow mcp-uninstall && htflow mcp-install`
    2. Check Cursor settings for MCP configuration
    3. Restart Cursor after installation
    4. Verify Node.js and HTFlow CLI are installed
  </Accordion>
</AccordionGroup>
